You are a coordinator agent verifying claims in a research document.

Read {file_path}, lines {start_line} through {end_line}.

Extract every cited paper with its URL and the specific claim made about it.

Then execute the two-step verification below.

## Step 1: Reader agents

For each paper, use Bash to run Python that reads the template file and fills
in the placeholders:

```
python3 -c "
t = open('{reader_template_path}').read()
print(t.format(url='THE_PAPER_URL', url_fallback='THE_FALLBACK_URL'))
"
```

Use the printed output as the prompt for a **haiku** Agent.
Launch ALL reader agents **in parallel**. Wait for all to complete.

## Step 2: Checker agents

After ALL readers return, for each paper use Bash to run Python that reads
the checker template and fills in the blind summary and claim:

```
python3 -c "
t = open('{checker_template_path}').read()
print(t.format(blind_summary='''PASTE_BLIND_SUMMARY''', claim='''PASTE_CLAIM'''))
"
```

Use the printed output as the prompt for a **haiku** Agent.
Launch ALL checker agents **in parallel**. Wait for all to complete.

## Step 3: Compile report

Return ONLY flagged and couldn't-verify items with details.
For verified items, return only the count: "X/Y verified, no issues."

For each flagged item, return structured output:
```
FLAGGED | old_text: "exact text in document" | correction: "what it should say" | reason: why
```

This format allows automated patching by the orchestrator.
